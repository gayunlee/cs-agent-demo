## 알려진 drift (매니저 현행 ↔ 저장된 템플릿)

| 템플릿 | 차이 | 조치 |
|---|---|---|
| **T2_환불_규정_금액** | 매니저 현행은 "1개월 정가 **의 1/3** 차감" + "(근거 부기)" + 규정 글머리 리스트 포함. 우리 템플릿은 간소 버전. | shadow pull → 매니저 현행 매크로 클러스터링 → 이 문서 업데이트 → 코드 sync |

다른 템플릿은 현재 드리프트 확인 안 됨 (수동 검수 필요).

## 템플릿 관리 정책

### 1. Single Source of Truth = 이 문서
- 코드 `src/templates.py`, `domain/templates.yaml`는 이 문서에서 파생된 사본
- 템플릿 변경 시 **반드시 이 문서 먼저 수정 → 코드 sync**
- 코드에만 변경이 들어가는 것은 금지 (drift 원인)

### 2. Drift 감지 (shadow pull 파이프라인)
- 매일 채널톡에서 매니저 실답변 pull
- 템플릿별로 클러스터링 (첫 2-3줄 유사도 or 해시)
- 이 문서의 원문과 diff → 차이 감지 시 alert
- Gayoon이 검수 후 이 문서 업데이트 → 코드 sync

### 3. 신규 템플릿 추가 절차
1. 이 문서에 새 섹션 작성 (ID, 변수, API, 원문)
2. `src/templates.py`에 entry 추가
3. `domain/templates.yaml`에 미러링
4. `domain/refund_chains.yaml`에 라우팅 규칙 추가
5. 테스트 케이스 작성 (`data/mock_scenarios/golden/v2/`)
6. 회귀 돌려서 다른 template 안 깨졌는지 확인

### 4. 변수 명명 규칙
